Lanyon Settings

This site is running on a modified version of the Lanyon theme - modifications are:

  • Sidebar is defaulted to open by editing sidebar.html

      <input type="checkbox" class="sidebar-checkbox" id="sidebar-checkbox" checked>

  • Default layout has the sidebar position swapped using

      <body class="layout-reverse sidebar-overlay">
  • public/js/script.js is commented out - this means that clicking on the screen doesn’t minimise the sidebar (it made clicking on links minimise then instantly maximise the sidebar, as sidebar is defaulted to open).

  • On line 83 of public/css/lanyon.css the max-width field upped from 38 to 56.

    @media (min-width: 56em) {
.container {
  max-width: 56rem;
}

jekyll-scholar

jekyll-scholar is used to manage references using a familiar .bib files and liquid tags. You need a .csl file to define how you want your references to render - this file goes in the assets folder. I am using a modified version of the ieee style.

CSL files

Citation Style Language is an XML-esque language used to define bibliography formatting. I have commented out part that says “Online” as it clutters up the rendered references.

 <macro name="access">
    <choose>
      <if variable="URL">
        <group delimiter=". ">
          <group delimiter=": ">
            <!-- <text term="available at" text-case="capitalize-first"/> -->
            <text variable="URL"/>
          </group>
          <group prefix="[" suffix="]" delimiter=": ">
            <text term="accessed" text-case="capitalize-first"/>
            <date variable="accessed">
              <date-part name="day" form="numeric-leading-zeros" suffix="-"/>
              <date-part name="month" form="short" suffix="-" strip-periods="true"/>
              <date-part name="year" form="long"/>
            </date>
          </group>
        </group>
      </if>
    </choose>
  </macro>

I was hoping to be able to inject raw HTML into this to support hyperlinks, but it looks like the rendering engine encodes the HTML code.

Hyperlinks in the URL

I wanted to have any URL fields in my bibliography render as hyperlinks. I tried editing the above CSL, as well as a monkey-patch, but neither worked.

I found that by adding a latex filter to the jekyll-scholar config in the root _config file

scholar:
  style: assets/bibliography/ieee-with-url.csl
  bibtex_filters:
    - latex

I was then able to write raw html in my .bib file, i.e.

@online{bashspecialvar,
url = {<a href="https://www.gnu.org/software/bash/manual/html_node/Special-Parameters.html">https://www.gnu.org/software/bash/manual/html_node/Special-Parameters.html</a>},
addendum = "(accessed: 01.01.2021)",
keywords = "bash, linux"
}

which then rendered the url field as a hyperlink. Whilst this works, it means the .bib file isn’t really usable outside of Jekyll.
