Monkey Patching

Overview

If you want to add functionality to a plugin without recompiling and including it yourself, you can simply create a Ruby file in the _plugins that monkey-patches whatever methods need changing.

Example [1]

I wanted bibliography references generated by jekyll-scholar to generate a hyperlink in the URL field. I found this [1] issue with a solution outlined.

module Jekyll
  class Scholar
    module Utilities

      alias_method :original_reference_tag, :reference_tag

      def reference_tag(entry)
        return missing_reference unless entry

        # Change URLs into markdown links
        entry["url"] = "[#{entry["url"]}](#{entry["url"]})" if entry["url"]

        entry = entry.convert(*bibtex_filters) unless bibtex_filters.empty?
        reference = CiteProc.process entry.to_citeproc,
          :style => style, :locale => config['locale'], :format => 'html'

        content_tag reference_tagname, reference,
          :id => [prefix, entry.key].compact.join('-')
      end
    end
  end
end

You can find the method as defined in jekyll-scholar here. The code above:

  • Aliases the reference_tag method that jekyll-scholar provides as old-reference-tag - meaning it won’t be recognised by the internal jekyll-scholar parsers.
  • Redefines the method, finding entry fields of url type and converting them to hyperlinks using markdown.

By putting this code in _plugins, you will get the functionality defined in the above snippet.

(This actually didn’t work for me, I had to find another workaround to get hyperlinks, but I think the snippet is a good example)
